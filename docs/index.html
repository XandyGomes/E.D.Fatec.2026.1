Algoritmos de Busca

A habilidade de encontrar informações de forma rápida é o que separa um software comum de um excelente.

Busca Sequencial vs. Binária

A Busca Sequencial percorre item por item. Imagine procurar uma palavra num dicionário começando da primeira página até a última. É simples, mas ineficiente para grandes volumes (O(n)).

Já a Busca Binária exige que os dados estejam ordenados. Ela corta o espaço de busca pela metade a cada passo.

Comparação visual entre Busca Sequencial e Binária.

Dica: Em Java, se você estiver usando um ArrayList, ele deve estar ordenado via Collections.sort() antes de tentar uma busca binária.
Algoritmos de Ordenação

Organização de Dados

Colocar a casa em ordem é o primeiro passo para o processamento de alto nível.

Estratégias de Comparação (Animações)

Bubble Sort: Os maiores elementos "flutuam" para o fim.

Selection Sort: Seleciona o menor elemento e o coloca na posição correta.

Merge Sort: Dividir para conquistar.

Quick Sort: Ordenação por particionamento rápido.

Lineares & Dinâmicas

Pilha e Fila

Como o fluxo de dados é controlado dentro da memória do computador.

Pilha (Stack) - LIFO

O último a entrar é o primeiro a sair. Essencial para controle de chamadas e histórico.

Fila (Queue) - FIFO

O primeiro a chegar é o primeiro a ser atendido. Usado em escalonamento e buffers.

Na Prática: O Deque no Java é implementado pela classe ArrayDeque ou LinkedList e permite o comportamento de Pilha e Fila simultaneamente.
Dinâmicas

Listas Encadeadas

Alocação dinâmica e flexível de memória.

Variantes de Listas

Diferente dos arrays, as Listas não ocupam espaços contínuos. Cada elemento (nodo) aponta para o próximo através de referências.

  • Simplesmente Encadeada: Cada nó aponta apenas para o próximo.
  • Duplamente Encadeada: Cada nó aponta para o próximo e para o anterior (facilita navegação inversa).
  • Circular: O último nó aponta de volta para o primeiro, criando um ciclo.
Vantagem: Inserções e remoções no início ou fim são extremamente rápidas (O(1)), pois não exigem o deslocamento de outros elementos.
Hierárquicas

Árvores de Busca (BST)

Estruturas não-lineares para buscas rápidas.

A Lógica Binária

Em uma Árvore Binária de Busca (BST), a regra é simples: para qualquer nó, todos os elementos à sua esquerda são menores e todos à direita são maiores.

Percursos em Árvores

Existem três formas clássicas de visitar todos os nós de uma árvore:

  • Pré-ordem (Pre-order): Raiz → Esquerda → Direita
  • Em-ordem (In-order): Esquerda → Raiz → Direita (Resulta na lista ordenada!)
  • Pós-ordem (Post-order): Esquerda → Direita → Raiz
Eficiência: Em uma árvore balanceada (como AVL ou Red-Black), as operações de busca, inserção e remoção custam apenas O(log n).
